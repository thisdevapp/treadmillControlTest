# 트레드밀 및 수조 제어 시스템 (S&D MediCare)

ESP32-C3와 USB 시리얼 통신을 통해 트레드밀과 수조를 제어하고 모니터링하는 Flutter 기반의 통합 제어 시스템입니다. 타블렛과 스마트폰 환경 모두에 최적화된 반응형 대시보드를 제공합니다.

## 🚀 주요 기능

### 1. 반응형 제어 대시보드
- **방향 인지 레이아웃**: 기기의 방향에 따라 가로 모드(좌우 2단 배치)와 세로 모드(상하 수직 배치)로 자동 전환됩니다.
- **머티리얼 3 디자인**: 시인성이 뛰어난 다크 테마와 현대적인 디자인을 적용하였습니다.
- **동적 스케일링**: 화면 크기와 방향에 맞춰 주요 수치(속도, 시간 등)의 폰트 크기와 컨트롤 배치가 유동적으로 조절됩니다.

### 2. 트레드밀 및 수조 관리
- **트레드밀 제어**: 
    - 속도(km/h), 시간, 이동 거리 실시간 모니터링.
    - 목표 속도(M/h) 및 타이머(분 단위) 설정 가능.
    - 물리적 시작/정지 명령 및 고성능 **긴급 정지(Emergency Stop)** 버튼 지원.
- **수조 제어**:
    - 물 온도(°C) 및 수위(cm) 실시간 모니터링.
    - 목표 온도 설정 (20°C ~ 45°C).
    - 물 받기(Fill Tank) 및 배수(Drain) 전용 제어 기능.

### 3. 고급 USB 시리얼 통신
- **프로토콜**: `*` 구분자를 사용한 견고한 명령 파싱 (예: `start*`, `setSpeed:150*`).
- **자동 장치 인식**: ESP32-C3(VID: 0x303A, PID: 0x1001) 연결 시 앱이 자동으로 인식되도록 인텐트 필터가 설정되어 있습니다.
- **높은 신뢰성**: 조각난 데이터 스트림 처리 및 다중 상태 업데이트 동시 처리가 가능합니다.

### 4. 디버깅 및 로그 시스템
- **로컬 저장**: 모든 UART 통신 내역과 시스템 이벤트를 밀리초 단위 타임스탬프와 함께 `app_debug_log.txt`에 저장합니다.
- **인앱 로그 터미널**: 앱 내에서 실시간으로 로그를 확인할 수 있는 로그 뷰어 오버레이를 제공합니다.
- **로그 추출**: 안드로이드 스튜디오의 Device File Explorer를 통해 사후 분석용 로그를 PC로 가져올 수 있습니다.

## 🛠 기술 설정

### 안드로이드 설정
- **NDK 버전**: 시리얼 플러그인 호환성을 위해 `27.0.12077973` 버전으로 명시적 설정.
- **권한 및 필터**: USB 호스트 기능 활성화 및 `AndroidManifest.xml` 내 하드웨어 필터 구성.

### 사용된 주요 패키지
- `usb_serial`: 핵심 시리얼 통신 라이브러리.
- `path_provider`: 로컬 파일 시스템 로그 저장.
- `intl`: 정밀한 타임스탬프 포맷팅.

## 📂 로그 파일 추출 방법
분석을 위해 기기에서 로그를 추출하는 방법입니다:
1. 안드로이드 기기를 PC에 연결합니다.
2. **Android Studio** 실행 -> **Device File Explorer**를 엽니다.
3. 다음 경로로 이동합니다: `data/data/com.example.testapp/app_flutter/app_debug_log.txt`.
4. 해당 파일에서 우클릭 후 **Save As**를 선택하여 PC에 저장합니다.
